# 机试题型样例

## Web安全实战

目标：对模拟电商网站进行渗透测试，获取管理员权限并提取flag。

任务：

1. 利用SQL注入获取后台管理员账号密码；
2. 通过JWT伪造登录后台；
3. 上传Webshell并执行系统命令；
4. 从数据库或文件系统中找到flag并提交。

## 二进制漏洞利用（Pwn）

目标：分析提供的ELF二进制文件，利用栈溢出漏洞获取shell。

任务：

1. 使用checksec查看保护机制；
2. 逆向分析漏洞函数；
3. 构造ROP链绕过NX/ASLR；
4. 获取flag并提交。

## 逆向工程与算法还原（Reverse）

目标：逆向分析一个授权验证程序，编写注册机。

任务：

1. 使用IDA/Ghidra进行静态分析；
2. 定位关键验证函数；
3. 还原算法逻辑；
4. 编写脚本生成有效序列号。

## 密码学算法破解（Crypto）

目标：破解一个自定义加密通信协议。

任务：

1. 分析通信流量；
2. 识别加密模式与密钥生成方式；
3. 编写解密脚本；
4. 提取flag。

## AI模型安全（新兴领域）

目标：对目标图像分类模型（例如识别“猫”或“狗”的CNN模型）进行白盒攻击，生成一张对抗样本图像，并获取隐藏的Flag。

任务：

1. 分析提供的模型结构文件（如 model.pth 或 model.h5），确定其输入维度与输出类别。
2. 使用FGSM或PGD等对抗攻击算法，对一张指定的原始图片（如“cat.jpg”）添加微小扰动。
3. 生成的对抗样本必须使模型以高置信度（如 >95%）将其错误分类为“狗”。
4. 成功欺骗模型后，系统会生成一个唯一的Flag，格式为 flag{...}。
5. 提交内容：
（1）最终获取的Flag字符串。
（2）所编写的攻击脚本源代码。
（3）对抗样本图片文件（用于验证视觉不可区分性）。

### PNG文件

- IHDR 文件头 `89 50 4E 47 0D 0A 1A 0A`
- IEND 文件结束 `00 00 00 00`
- 数据标识 `49 45 4E 44`
- CRC `AE 42 60 82`
- 因此PNG一般固定以 `00 00 00 00 49 45 4E 44 AE 42 60 82`作为结束

### ZIP文件

- 开始标记 `50 4B 03 04`

### Binwalk

- 开源固件分析工具
- 用于根据各类文件的特征分析一个文件并从一个文件中提取其他文件

### LSB隐写

- LSB指最低有效位
- (255, 0 ,0) -> (254, 0, 0) = 0, 0, 0
- (255, 0 ,0) -> (254, 1, 1) = 0, 1, 1
- (255, 0 ,0) -> (255, 0, 1) = 1, 0, 1

- "CTF" -> ASCII = 010 000 11 0 101 010 0 01 000 110
隐藏在8个像素中
1. (0, 0, 255) (51, 102, 255) (102, 102, 255) (153, 102, 255) (204, 102, 255) (255, 0, 255) (204, 0, 153) (153, 0, 153)
2. (0, 1, 254) (50, 102, 254) (103, 103, 254) (153, 102, 255) (204, 103, 254) (254, 0, 255) (204, 0, 152) (153, 1, 152)

> 通常用Stegsolve / zsteg 检测LSB隐写痕迹
